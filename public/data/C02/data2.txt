Pregunta 1: **Omitido¿Qué herramienta te ayudará a revisar tus cargas de trabajo comparándolas con las mejores prácticas actuales de AWS en cuanto a optimización de costes, seguridad y mejora del rendimiento y, a continuación, a obtener asesoramiento para diseñarlas mejor?**
• **AWS Cost Explorer**
• **Amazon Inspector**
• **AWS Trusted Advisor(Correcto)**
• **Amazon CloudWatch
Explicación**
Opción correcta:
**AWS Trusted Advisor**
AWS Trusted Advisor es una herramienta online que te proporciona orientación en tiempo real para ayudarte a aprovisionar tus recursos siguiendo las mejores prácticas de AWS sobre optimización de costes, seguridad, tolerancia a fallos, límites de servicio y mejora del rendimiento. Ya sea para establecer nuevos flujos de trabajo, desarrollar aplicaciones o como parte de una mejora continua, las recomendaciones proporcionadas regularmente por Trusted Advisor ayudan a mantener tus soluciones aprovisionadas de forma óptima. Todos los clientes de AWS obtienen acceso a las siete comprobaciones básicas de Trusted Advisor para ayudar a aumentar la seguridad y el rendimiento del entorno de AWS.Cómo funciona Trusted Advisor:  vía - https://aws.amazon.com/premiumsupport/technology/trusted-advisor/Recomendaciones de AWS Trusted Advisor:  vía - https://aws.amazon.com/premiumsupport/technology/trusted-advisor/
Opciones incorrectas:
**AWS Cost Explorer** - AWS Cost Explorer te permite explorar tus costes y uso de AWS tanto a un alto nivel como a un nivel detallado de análisis, y te faculta para profundizar utilizando varias dimensiones de filtrado (por ejemplo, servicio de AWS, región, cuenta vinculada). AWS Cost Explorer no ofrece recomendaciones sobre las mejores prácticas de AWS para la optimización de costes, la seguridad y la mejora del rendimiento.
**Amazon CloudWatch** - Amazon CloudWatch es un servicio de monitorización y observabilidad creado para ingenieros DevOps, desarrolladores, ingenieros de fiabilidad de sitios (SRE) y administradores de TI. CloudWatch proporciona datos y perspectivas procesables para monitorizar aplicaciones, responder a cambios de rendimiento en todo el sistema, optimizar la utilización de recursos y obtener una visión unificada de la salud operativa. Es un servicio excelente para construir sistemas resilientes. Piensa en supervisión del rendimiento de los recursos, eventos y alertas; piensa en CloudWatch. CloudWatch no ofrece recomendaciones sobre las mejores prácticas de AWS para la optimización de costes, la seguridad y la mejora del rendimiento.
**Amazon Inspector** - Amazon Inspector es un servicio automatizado de evaluación de la seguridad que ayuda a mejorar la seguridad y la normativa de las aplicaciones implementadas en AWS. Amazon Inspector evalúa automáticamente las aplicaciones en busca de exposición, vulnerabilidades y desviaciones de las mejores prácticas. Amazon Inspector no puede utilizarse para evitar ataques de denegación de servicio distribuido (DDoS). Inspector no ofrece recomendaciones respecto a las mejores prácticas de AWS para la optimización de costes, la seguridad y la mejora del rendimiento.

Pregunta 3: **Omitido¿Cuál de las siguientes afirmaciones es correcta en relación con la política de precios de AWS para los cargos por transferencia de datos dentro o fuera de una región de AWS?**

• **Sólo se cobra la transferencia de datos salientes(Correcto)**
• **Sólo se cobra la transferencia de datos entrantes

Opción correcta:

Pregunta 4: **OmitidoSegún el modelo de responsabilidad compartida de AWS, ¿cuál de las siguientes es una responsabilidad de AWS desde el punto de vista de la seguridad y la normativa?**
• **Datos del cliente**
• **Gestión de Edge Locations(Correcto)**
• **Encriptación del lado del servidor (SSE)**
• **Identity and Access Management (IAM)
Explicación**
Opción correcta:
**Gestión de Edge Locations**
La seguridad y la normativa son una responsabilidad compartida entre AWS y el cliente. El modelo de responsabilidad compartida de AWS puede ayudar a aliviar la carga operativa del cliente, ya que AWS opera, administra y controla los componentes, desde el sistema operativo anfitrión y la capa de virtualización hasta la seguridad física de las instalaciones en las que opera el servicio.
AWS es responsable de la seguridad "del" Cloud. Esto abarca sus elementos de infraestructura global, incluidas las regiones, las zonas de disponibilidad (AZ) y las Ubicaciones de Borde.
Opciones incorrectas:
**Datos del cliente**
**Identity and Access Management (IAM)**
**Encriptación del lado del servidor (SSE)**
El cliente es responsable de la seguridad "en" el Cloud. Los clientes son responsables de gestionar sus datos, incluidas las opciones de cifrado, y de utilizar herramientas de Gestión de Identidad y Acceso para aplicar políticas de control de acceso adecuadas a los requisitos de su organización. Para los servicios abstraídos, como Amazon S3 y Amazon DynamoDB, AWS opera la capa de infraestructura, el sistema operativo y las plataformas, y los clientes acceden a los puntos finales para almacenar y recuperar datos. Por tanto, estas tres opciones caen bajo la responsabilidad del cliente según el modelo de responsabilidad compartida de AWS.

Pregunta 6: **Omitido¿Cuál es la principal ventaja de desplegar una base de datos Multi-AZ de Amazon RDS con un standby?**

• **Amazon RDS Multi-AZ mejora el rendimiento de la base de datos para cargas de trabajo de lectura intensiva**
• **Amazon RDS Multi-AZ reduce los costes de uso de la base de datos**
• **Amazon RDS Multi-AZ mejora la disponibilidad de la base de datos(Correcto)
Explicación**
Opción correcta:
**Amazon RDS Multi-AZ mejora la disponibilidad de la base de datos**
Los despliegues Multi-AZ de Amazon RDS proporcionan una mayor disponibilidad y durabilidad para las instancias de Amazon Relational Database Service (Amazon RDS), lo que las convierte en una opción natural para las cargas de trabajo de bases de datos de producción. Cuando aprovisionas una instancia Multi-AZ de Amazon RDS con una instancia en espera, Amazon RDS crea automáticamente una instancia de base de datos primaria y replica de forma síncrona los datos a una instancia en espera en una zona de disponibilidad (AZ) diferente.
En caso de fallo de la infraestructura, Amazon RDS realiza una conmutación por error automática al sistema en espera para que puedas reanudar las operaciones de la base de datos en cuanto se complete la conmutación por error.Cómo funciona Amazon RDS Multi-AZ:  vía - https://aws.amazon.com/rds/features/multi-az/
Alerta de examen:Por favor, revisa las diferencias entre los despliegues multi-AZ, multi-región y réplica de lectura para RDS:  vía - https://aws.amazon.com/rds/features/multi-az/
Opciones incorrectas:
**Amazon RDS Multi-AZ mejora el rendimiento de la base de datos para cargas de trabajo de lectura intensiva** - Amazon RDS multi-AZ con una base en espera no permite operaciones de lectura desde la base en espera. Las réplicas de lectura te permiten crear copias de sólo lectura que se sincronizan con tu base de datos maestra. Las réplicas de lectura se utilizan para mejorar el rendimiento de lectura. Por tanto, esta opción es incorrecta.
**Amazon RDS Multi-AZ protege la base de datos de un fallo regional** - Tienes que utilizar RDS en configuración de despliegue multi-región para protegerte de un fallo regional. Amazon RDS multi-AZ no puede proteger de un fallo regional.
**Amazon RDS Multi-AZ reduce los costes de uso de la base de datos** - Amazon RDS multi-AZ aumenta los costes de la base de datos en comparación con el despliegue estándar. Por tanto, esta opción es incorrecta.

Pregunta 9: **Omitido¿Cuál de los siguientes servicios de AWS puede utilizarse para evitar un ataque de denegación de servicio distribuido (DDoS)? (Selecciona tres)**

Explicación**
Opciones correctas:
**AWS Shield** - AWS Shield es un servicio gestionado de protección contra la Denegación de Servicio Distribuida (DDoS) que protege las aplicaciones que se ejecutan en AWS. AWS Shield proporciona detección siempre activa y mitigaciones automáticas en línea que minimizan el tiempo de inactividad y la latencia de las aplicaciones, por lo que no es necesario contratar a AWS Support para beneficiarse de la protección DDoS.
**AWS Web Application Firewall (AWS WAF)** - Utilizando AWS Web Application Firewall (AWS WAF), puedes configurar listas de control de acceso web (ACL web) en tus distribuciones de Amazon CloudFront o Application Load Balancers para filtrar y bloquear solicitudes en función de las firmas de las solicitudes. Además, utilizando las reglas basadas en la tasa de AWS Web Application Firewall (AWS WAF), puedes bloquear automáticamente las direcciones IP de los malos actores cuando las solicitudes que coincidan con una regla superen un umbral que tú definas.
**Amazon CloudFront con Amazon Route 53** - AWS aloja los servicios Amazon CloudFront y Amazon Route 53 en una red distribuida de servidores proxy en centros de datos de todo el mundo denominados ubicaciones de borde (Edge Locations). El uso de la red global de Amazon de ubicaciones de borde para la entrega de aplicaciones y el servicio DNS desempeña un papel importante en la construcción de una defensa integral contra ataques DDoS para tus aplicaciones web dinámicas.Cómo AWS Shield, AWS Web Application Firewall (AWS WAF) y Amazon CloudFront con Amazon Route 53 ayudan a mitigar los ataques DDoS:  vía - https://aws.amazon.com/blogs/security/how-to-protect-dynamic-web-applications-against-ddos-attacks-by-using-amazon-cloudfront-and-amazon-route-53/

**AWS CloudHSM** - AWS CloudHSM es un módulo de seguridad de hardware (HSM) basado en el Cloud que te permite generar y utilizar fácilmente tus claves de cifrado en el Cloud de AWS. Con CloudHSM, puedes gestionar tus claves de cifrado utilizando HSM validados por FIPS 140-2 Nivel 3. Se trata de un servicio totalmente gestionado que automatiza por ti las tareas administrativas que requieren mucho tiempo, como el aprovisionamiento de hardware, la aplicación de parches de software, la alta disponibilidad y las copias de seguridad. CloudHSM no puede utilizarse para evitar ataques distribuidos de denegación de servicio (DDoS).
**AWS Trusted Advisor** - AWS Trusted Advisor es una herramienta online que te proporciona orientación en tiempo real para ayudarte a aprovisionar tus recursos siguiendo las mejores prácticas de AWS sobre optimización de costes, seguridad, tolerancia a fallos, límites de servicio y mejora del rendimiento. Ya sea estableciendo nuevos flujos de trabajo, desarrollando aplicaciones o como parte de una mejora continua, las recomendaciones proporcionadas regularmente por Trusted Advisor ayudan a mantener tus soluciones aprovisionadas de forma óptima. Trusted Advisor no puede utilizarse para evitar ataques de Denegación de Servicio Distribuido (DDoS).
**Amazon Inspector** - Amazon Inspector es un servicio automatizado de evaluación de la seguridad que ayuda a mejorar la seguridad y la normativa de las aplicaciones implementadas en AWS. Amazon Inspector evalúa automáticamente las aplicaciones en busca de exposición, vulnerabilidades y desviaciones de las mejores prácticas. Amazon Inspector no puede utilizarse para evitar ataques de denegación de servicio distribuidos (DDoS).

Pregunta 61: **Omitido¿Cuál de los siguientes servicios de AWS pertenece al tipo de Cloud Computing Software as a Service (SaaS)?**
• **Amazon Rekognition(Correcto)**
• **Equilibrio de carga elástico (ELB)**
• **Amazon Elastic Compute Cloud (Amazon EC2)**
• **AWS Elastic Beanstalk
Explicación**
Opción correcta:
**Amazon Rekognition**
El Cloud Computing puede dividirse a grandes rasgos en tres tipos: infraestructura como servicio (IaaS), plataforma como servicio (PaaS) y software como servicio (SaaS).
La IaaS contiene los componentes básicos de la TI en el Cloud. Suele proporcionar acceso a funciones de red, ordenadores (virtuales o en hardware dedicado) y espacio de almacenamiento de datos. La IaaS ofrece el máximo nivel de flexibilidad y control de gestión sobre los recursos informáticos. Ejemplos: Amazon EC2 (en AWS), GCP, Azure, Rackspace, Digital Ocean, Linode.
PaaS elimina la necesidad de gestionar la infraestructura subyacente (normalmente hardware y sistemas operativos) y te permite centrarte en el despliegue y la gestión de tus aplicaciones. No tienes que preocuparte por la obtención de recursos, la planificación de la capacidad, el mantenimiento del software, los parches ni ninguna de las otras tareas pesadas e indiferenciadas que implica el funcionamiento de tu aplicación. Ejemplos - Elastic Beanstalk (en AWS), Heroku, Google App Engine (GCP), Windows Azure (Microsoft).
SaaS te proporciona un producto completo que dirige y gestiona el proveedor de servicios. Con una oferta SaaS, no tienes que pensar en cómo se mantiene el servicio o cómo se gestiona la infraestructura subyacente. Sólo tienes que pensar en cómo vas a utilizar ese software concreto. Ejemplos: Amazon Rekognition, Google Apps (Gmail), Dropbox, Zoom.Visión general de los tipos de Cloud Computing:  vía - https://aws.amazon.com/types-of-cloud-computing/
Puedes utilizar Amazon Rekognition para añadir análisis de imágenes y vídeos a tus aplicaciones utilizando tecnología de aprendizaje profundo probada, altamente escalable y que no requiere conocimientos de aprendizaje automático. Con Amazon Rekognition, puedes identificar objetos, personas, texto, escenas y actividades en imágenes y vídeos, así como detectar cualquier contenido inapropiado. Rekognition es un ejemplo de modelo de Software como Servicio (Saas).
Opciones incorrectas:
**Amazon Elastic Compute Cloud (Amazon EC2)** - Amazon Elastic Compute Cloud (Amazon EC2) es un servicio web que proporciona capacidad informática segura y redimensionable en el Cloud. Por tanto, pertenece al tipo de Cloud Computing de Infraestructura como Servicio (IaaS).

Sólo tienes que subir tu código y Elastic Beanstalk se encarga automáticamente del despliegue, desde el aprovisionamiento de capacidad, el equilibrio de carga y el autoescalado hasta la monitorización del estado de la aplicación. Según las definiciones anteriores, Elastic Beanstalk pertenece al tipo plataforma como Servicio (PaaS).
**Equilibrio de carga elástico (ELB)** - Elastic Load Balancer (ELB) distribuye automáticamente el tráfico entrante de la aplicación entre múltiples destinos, como instancias de Amazon EC2, contenedores, direcciones IP y funciones Lambda. Puede gestionar la carga variable del tráfico de tu aplicación en una única zona de disponibilidad (AZ) o en varias zonas de disponibilidad (AZ). Esto se ha añadido como distractor.

Pregunta 1: **Correcto¿Cuál de las siguientes es una forma recomendada de proporcionar acceso programático a los recursos de AWS?**
• **Utiliza el ID de la clave de acceso y la clave de acceso secreta para acceder a los recursos de AWS mediante programación(Correcto)**
• **Utiliza el grupo de usuarios IAM para acceder a los recursos de AWS mediante programación**
• **Crear un nuevo usuario IAM y compartir el nombre de usuario y la contraseña**
• **Utiliza AWS Multi-Factor Authentication (AWS MFA) para acceder a los recursos de AWS mediante programación
Explicación**
Opción correcta:
**Utiliza el ID de la clave de acceso y la clave de acceso secreta para acceder a los recursos de AWS mediante programación**
Las claves de acceso son credenciales a largo plazo para un usuario de IAM o el usuario raíz de la cuenta de AWS. Puedes utilizar claves de acceso para firmar solicitudes programáticas a la CLI de AWS o a la API de AWS (directamente o utilizando el SDK de AWS). Las claves de acceso constan de dos partes: un ID de clave de acceso y una clave de acceso secreta. Como nombre de usuario y contraseña, debes utilizar conjuntamente el ID de la clave de acceso y la clave de acceso secreta para autenticar tus solicitudes. Cuando crees un par de claves de acceso, guarda el ID de la clave de acceso y la clave de acceso secreta en un lugar seguro. La clave de acceso secreta sólo está disponible en el momento en que la creas. Si pierdes la clave de acceso secreta, debes borrar la clave de acceso y crear una nueva.
Opciones incorrectas:
**Crear un nuevo usuario IAM y compartir el nombre de usuario y la contraseña** - No es una opción viable, las credenciales de usuario IAM no son necesarias para acceder a los recursos mediante programación.
**Utiliza AWS Multi-Factor Authentication (AWS MFA) para acceder a los recursos de AWS mediante programación** - Para aumentar la seguridad, AWS recomienda que configures AWS Multi-Factor Authentication (AWS MFA) para ayudar a proteger tus recursos de AWS. Puedes habilitar MFA para los usuarios de IAM o para el usuario raíz de la cuenta de AWS. MFA añade seguridad adicional porque requiere que los usuarios proporcionen una autenticación única de un mecanismo MFA compatible con AWS, además de sus credenciales de inicio de sesión habituales, cuando acceden a los sitios web o servicios de AWS. MFA no puede utilizarse para el acceso programático a los recursos de AWS.
**Utiliza el grupo de usuarios IAM para acceder a los recursos de AWS mediante programación** - Un grupo de usuarios IAM es un conjunto de usuarios IAM. Un grupo de usuarios IAM te permite especificar permisos para varios usuarios, lo que puede facilitar la gestión de los permisos de esos usuarios. El grupo de usuarios IAM sirve para gestionar usuarios y no para programar el acceso a los recursos de AWS.
Referencia:
https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html

Pregunta 2: **Omitido¿Cuál de las siguientes clases de almacenamiento de Amazon S3 tarda más en recuperar datos (también conocida como latencia del primer byte)?**
• **Amazon S3 Standard**
• **Amazon S3 Glacier Deep Archive(Correcto)**
• **Amazon S3 Glacier**
• **Amazon S3 Intelligent-Tiering
Explicación**
Opción correcta:
**Amazon S3 Glacier Deep Archive**
Amazon S3 Glacier Deep Archive es la clase de almacenamiento de menor coste de Amazon S3 y soporta la retención a largo plazo y la preservación digital de datos a los que se puede acceder una o dos veces al año. Está diseñado para clientes -especialmente los de sectores muy regulados, como los servicios financieros, la sanidad y el sector público- que conservan conjuntos de datos durante 7-10 años o más para cumplir los requisitos normativos. S3 Glacier Deep Archive también puede utilizarse para casos de uso de backup y recuperación de desastres. Tiene un tiempo de recuperación (latencia del primer byte) de 12 a 48 horas.Revisa esta ilustración para conocer los tiempos de recuperación de datos de las clases de almacenamiento de Amazon S3. No necesitas memorizar los números reales, sólo recuerda que S3 Glacier Deep Archive es el que tarda más tiempo en recuperar los datos:  vía - https://aws.amazon.com/s3/storage-classes/
Opciones incorrectas:
**Amazon S3 Standard** - Amazon S3 Standard ofrece almacenamiento de objetos de alta durabilidad, disponibilidad y rendimiento para datos a los que se accede con frecuencia. Amazon S3 Standard tiene un tiempo de recuperación (latencia del primer byte) de milisegundos.
**Amazon S3 Intelligent-Tiering** - La clase de almacenamiento Amazon S3 Intelligent-Tiering está diseñada para optimizar los costes moviendo automáticamente los datos al nivel de acceso más rentable, sin impacto en el rendimiento ni sobrecarga operativa. Funciona almacenando objetos en dos niveles de acceso: un nivel optimizado para el acceso frecuente y otro nivel de menor coste optimizado para el acceso poco frecuente. Amazon S3 Intelligent-Tiering tiene un tiempo de recuperación (latencia del primer byte) de milisegundos.
**Amazon S3 Glacier** - Amazon S3 Glacier Flexible Retrieval ofrece almacenamiento de bajo costo, hasta un 10% más barato (que Amazon S3 Glacier Instant Retrieval), para datos de archivo a los que se accede 1-2 veces al año y que se recuperan de forma asíncrona. Para los datos de archivo que no requieren acceso inmediato pero necesitan flexibilidad para recuperar grandes conjuntos de datos sin coste alguno, como los casos de uso de backup o recuperación de desastres, Amazon S3 Glacier Flexible Retrieval (antes Amazon S3 Glacier) es la clase de almacenamiento ideal.
Referencia:
https://aws.amazon.com/s3/storage-classes/

• **Host dedicado(Correcto)**
• **Instancia reservada (RI)**
• **Instancia bajo demanda

Opción correcta:
**Host dedicado**
Los hosts dedicados de Amazon EC2 te permiten utilizar tus licencias de software elegibles de proveedores como Microsoft y Oracle en Amazon EC2. Un host dedicado de Amazon EC2 es un servidor físico totalmente dedicado para tu uso, por lo que puedes ayudar a cumplir los requisitos corporativos.

Pregunta 9: **Omitido¿Cuál de las siguientes es una afirmación INCORRECTA sobre el Escalado, un principio de diseño del pilar fiabilidad del AWS Well-Architected Framework?**
• **La tolerancia a fallos se consigue mediante una operación de escalado horizontal**
• **La tolerancia a fallos se consigue mediante una operación de escalado vertical(Correcto)**
• **Una operación de escalado implica que escalas añadiendo más potencia (CPU, RAM) a tu máquina/nodo existente**
• **Una operación de ampliación implica que se amplía añadiendo más instancias al conjunto de recursos existente
Explicación**
Opción correcta:
**La tolerancia a fallos se consigue mediante una operación de escalado vertical**
Una operación a escala está limitada a ejecutar sus procesos en un solo ordenador. En tales sistemas, la única forma de aumentar el rendimiento es añadir más recursos a un ordenador en forma de CPU más rápidas, memoria o almacenamiento. La tolerancia a fallos no es posible en tales operaciones de escalado, ya que una sola instancia es propensa a fallar.
Opciones incorrectas:
**Una operación de escalado implica que escalas añadiendo más potencia (CPU, RAM) a tu máquina/nodo existente** - Una operación de ampliación se ejecuta en una única instancia. El aumento de potencia sólo es posible mediante la adición de recursos en forma de CPU, RAM o almacenamiento para mejorar el rendimiento.
**Una operación de ampliación implica que se amplía añadiendo más instancias al conjunto de recursos existente** - Una operación de ampliación es aquella que puede aumentar la capacidad añadiendo más ordenadores al sistema. Los sistemas scale out (escalado horizontal) suelen superar a los scale up (escalado vertical) al permitir la ejecución paralela de cargas de trabajo y distribuirlas entre muchos ordenadores diferentes.
**La tolerancia a fallos se consigue mediante una operación de escalado horizontal** - Una operación de ampliación añade más instancias a un conjunto de instancias existente. Esto implica que no hay un único punto de fallo. Si una instancia se cae, la carga de trabajo es asumida por otras instancias sanas. Los sistemas distribuidos son un ejemplo de este tipo de escalado.
Referencia:
https://wa.aws.amazon.com/wat.concept.horizontal-scaling.en.html

Amazon EFS es un servicio de almacenamiento de archivos para su uso con Amazon EC2. Amazon EFS proporciona una interfaz de sistema de archivos, semántica de acceso al sistema de archivos y almacenamiento accesible simultáneamente para hasta miles de instancias de Amazon EC2. **Configurar la base de datos en despliegue RDS Multi-AZ con conmutación por error automática a la base en espera**
Cuando aprovisionas una Instancia de Base de Datos Multi-AZ, Amazon RDS crea automáticamente una Instancia de Base de Datos primaria y replica de forma síncrona los datos a una instancia en espera en una zona de disponibilidad (AZ) diferente. En caso de fallo de la infraestructura, Amazon RDS realiza una conmutación por error automática a la instancia en espera (o a una réplica de lectura en el caso de Amazon Aurora), para que puedas reanudar las operaciones de la base de datos en cuanto se complete la conmutación por error. Como el punto final de tu Instancia de Base de Datos sigue siendo el mismo después de una conmutación por error, tu aplicación puede reanudar el funcionamiento de la base de datos sin necesidad de intervención administrativa manual.
Opciones incorrectas:
**Implementar la base de datos a través de AWS Elastic Beanstalk** - No puedes desplegar sólo una base de datos a través de Elastic Beanstalk, ya que está pensado para el despliegue automático de aplicaciones cuando subes tu código. Entonces Elastic Beanstalk gestiona automáticamente el despliegue, desde el aprovisionamiento de capacidad, el equilibrio de carga, el autoescalado hasta la supervisión del estado de la aplicación. Por lo tanto, esta opción es incorrecta.
**Configurar la base de datos en modo de réplica de lectura RDS con conmutación por error automática a la reserva** - Para RDS, las réplicas de lectura te permiten crear copias de sólo lectura que se sincronizan con tu base de datos maestra. Al utilizar réplicas de lectura no se dispone de standby. En caso de fallo de la infraestructura, tienes que promover manualmente la réplica de lectura para que sea su propia Instancia de BD autónoma, lo que significa que cambiaría el punto final de la base de datos. Por tanto, esta opción es incorrecta.
**Aprovisionar la base de datos mediante AWS CloudFormation** - Seguro que puedes aprovisionar la base de datos mediante CloudFormation, pero no proporciona ninguna recuperación automática en caso de desastre.

Pregunta 23: **Omitido¿Cuál de los siguientes servicios de AWS admite VPC Endpoint Gateway para una conexión privada desde una VPC? (Selecciona dos)**

Explicación**
Opciones correctas:
**Amazon Simple Storage Service (Amazon S3)**
**Amazon DynamoDB**
Un Endpoint de VPC (también llamado punto final) te permite conectar de forma privada tu VPC a los servicios de AWS compatibles y a los servicios de punto final de VPC impulsados por AWS PrivateLink sin necesidad de una puerta de enlace a Internet, un dispositivo NAT, una conexión VPN o una conexión AWS Direct Connect. Las instancias de tu VPC no necesitan direcciones IP públicas para comunicarse con los recursos del servicio. El tráfico entre tu VPC y el otro servicio no sale de la red de Amazon.
Hay dos tipos de endpoints de la VPC: endpoints de interfaz y endpoints de Gateway.
Un endpoint de interfaz es una interfaz de red elástica con una dirección IP privada del rango de direcciones IP de tu subred que sirve como punto de entrada para el tráfico destinado a un servicio soportado. Los puntos finales de interfaz funcionan con AWS PrivateLink, una tecnología que te permite acceder de forma privada a los servicios utilizando direcciones IP privadas.
Un Gateway Endpoint es una puerta de enlace que especificas como destino de una ruta en tu tabla de rutas para el tráfico destinado a un servicio de AWS soportado. Se admiten los siguientes servicios de AWS:
Amazon Simple Storage Service (Amazon S3)
Amazon DynamoDB
Alerta de examen:
Puede que veas una pregunta sobre este concepto en el examen. Recuerda que sólo Amazon S3 y Amazon DynamoDB soportan el Endpoint de Gateway de VPC. Todos los demás servicios que admiten puntos finales de VPC utilizan un punto final de interfaz de VPC (ten en cuenta que Amazon S3 también admite el endpoint de interfaz de VPC).
Opciones incorrectas:
**Amazon Elastic Compute Cloud (Amazon EC2)**
**Amazon Simple Queue Service (SQS)**
**Amazon Simple Notification Service (SNS)**
Como ya se ha explicado, estos servicios soportan las interfaces de endpoints de la VPC.

La seguridad y la normativa es una responsabilidad compartida entre AWS y el cliente. Una instancia reservada (RI) no es una instancia física, sino un descuento de facturación aplicado al uso de instancias bajo demanda en tu cuenta. Puedes adquirir una instancia reservada (RI) para un compromiso de uno o tres años, siendo el compromiso de tres años el que ofrece un mayor descuento. Las instancias spot son muy adecuadas para análisis de datos, trabajos por lotes, procesamiento en segundo plano y tareas opcionales. Pueden interrumpirse con poca antelación, por lo que no son adecuadas para cargas de trabajo críticas que deban ejecutarse en un momento determinado. Por tanto, esta opción no es correcta para el caso de uso planteado.

Un host dedicado de Amazon EC2 es un servidor físico totalmente dedicado para tu uso, de modo que puedes ayudar a cumplir los requisitos corporativos. No es rentable en comparación con una instancia bajo demanda. Por tanto, esta opción no es correcta.

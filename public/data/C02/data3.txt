Pregunta 1: **OmitidoComo Cloud Practitioner, ¿qué clase de almacenamiento de Amazon Simple Storage Service (Amazon S3) recomendarías para archivar datos?**
• **Amazon S3 Glacier Flexible Retrieval(Correcto)**
• **Amazon S3 Standard**
• **Amazon S3 Intelligent-Tiering**
• **Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA)
Explicación**
Opción correcta:
**Amazon S3 Glacier Flexible Retrieval**
Amazon S3 Glacier Flexible Retrieval (antes S3 Glacier) es la clase de almacenamiento ideal para archivar datos que no requieren acceso inmediato pero necesitan flexibilidad para recuperar grandes conjuntos de datos sin coste alguno, como los casos de uso de backup o recuperación de desastres. Es una clase de almacenamiento en el Cloud de Amazon S3 segura, duradera y de coste extremadamente bajo para archivar datos y realizar copias de seguridad a largo plazo. Está diseñado para ofrecer una durabilidad del 99,999999999%, y proporcionar capacidades integrales de seguridad y normativa que pueden ayudar a cumplir incluso los requisitos normativos más estrictos.Puedes revisar más a fondo los casos de uso de S3 Glacier Flexible Retrieval:  vía - https://aws.amazon.com/glacier/Visión general de las clases de almacenamiento S3:  vía - https://aws.amazon.com/s3/storage-classes/
Opciones incorrectas:
**Amazon S3 Standard** - Amazon S3 Standard ofrece almacenamiento de objetos de alta durabilidad, disponibilidad y rendimiento para datos a los que se accede con frecuencia. No es adecuado para archivar datos.
**Amazon S3 Intelligent-Tiering** - La clase de almacenamiento Amazon S3 Intelligent-Tiering está diseñada para optimizar los costes moviendo automáticamente los datos al nivel de acceso más rentable, sin impacto en el rendimiento ni sobrecarga operativa. Funciona almacenando objetos en dos niveles de acceso: un nivel optimizado para el acceso frecuente y otro nivel de menor coste optimizado para el acceso poco frecuente. No es adecuado para archivar datos.
**Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA)** - Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA) es para datos a los que se accede con menos frecuencia, pero que requieren un acceso rápido cuando se necesitan. A diferencia de otras clases de almacenamiento de S3 que almacenan datos en un mínimo de tres zonas de disponibilidad (AZ), Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA) almacena datos en una única zona de disponibilidad (AZ). No es adecuada para archivar datos.
Referencias:
https://aws.amazon.com/glacier/
https://aws.amazon.com/s3/storage-classes/

Pregunta 2: **Omitido¿Cuáles de las siguientes son las mejores prácticas recomendadas para el servicio AWS Identity and Access Management (AWS IAM)? (Selecciona dos)**
• [ ] **Rota las credenciales regularmente(Correcto)**
• [ ] **Crea un número mínimo de cuentas y comparte las credenciales de estas cuentas entre los empleados**
• [ ] **Comparte las claves de acceso al usuario raíz de la cuenta de AWS con otros administradores**
• [ ] **Concede privilegios máximos para evitar asignar privilegios de nuevo**
• [ ] **Activa la autenticación multifactor (MFA) para todos los usuarios(Correcto)
Explicación**
Opción correcta:
**Activa la autenticación multifactor (MFA) para todos los usuarios**
AWS recomienda que exijas la autenticación multifactor (MFA) a todos los usuarios de tu cuenta. Con la autenticación multifactor (MFA), los usuarios disponen de un dispositivo que genera una respuesta a un desafío de autenticación. Tanto las credenciales del usuario como la respuesta generada por el dispositivo son necesarias para completar el proceso de inicio de sesión.
**Rota las credenciales regularmente**
AWS recomienda que cambies tus propias contraseñas y claves de acceso con regularidad, y que te asegures de que todos los usuarios de IAM de tu cuenta también lo hagan. De ese modo, si una contraseña o clave de acceso se ve comprometida sin tu conocimiento, limitas el tiempo durante el que las credenciales pueden utilizarse para acceder a tus recursos. Puedes aplicar una política de contraseñas a tu cuenta para exigir a todos tus usuarios de IAM que roten sus contraseñas.Mejores prácticas de seguridad de AWS IAM:  vía - https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html
Opciones incorrectas:
**Crea un número mínimo de cuentas y comparte las credenciales de estas cuentas entre los empleados** - AWS recomienda que las credenciales de las cuentas de usuario no se compartan entre usuarios.
**Concede privilegios máximos para evitar asignar privilegios de nuevo** - AWS recomienda conceder los menores privilegios necesarios para completar un determinado trabajo y evitar dar privilegios excesivos que puedan ser mal utilizados.
**Comparte las claves de acceso al usuario raíz de la cuenta de AWS con otros administradores** - La clave de acceso del usuario raíz de tu cuenta de AWS da acceso completo a todos tus recursos para todos los servicios de AWS, incluida tu información de facturación. No puedes reducir los permisos asociados a la clave de acceso del usuario raíz de tu cuenta de AWS. Nunca debes compartir estas claves de acceso con ningún otro usuario, ni siquiera con los administradores.
Referencia:
https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html

Pregunta 3: **OmitidoSegún el modelo de responsabilidad compartida de AWS, ¿cuáles de las siguientes son las responsabilidades del cliente? (Selecciona dos)**
• [ ] **Validación de normativa de la infraestructura del Cloud**
• [ ] **Habilitación del cifrado de datos almacenados en buckets de Amazon Simple Storage Service (Amazon S3)(Correcto)**
• [ ] **Parches y actualizaciones del sistema operativo de una instancia de Amazon Elastic Compute Cloud (Amazon EC2)(Correcto)**
• [ ] **Garantizar que los empleados de AWS no puedan acceder a los datos de los clientes**
• [ ] **Seguridad global de la red de AWS
Explicación**
Opciones correctas:
Bajo el modelo de responsabilidad compartida de AWS, AWS es responsable de proteger la infraestructura que ejecuta todos los servicios ofrecidos en el Cloud de AWS. Esta infraestructura se compone del hardware, software, redes e instalaciones que ejecutan los servicios del Cloud de AWS. La responsabilidad del cliente viene determinada por los servicios en el Cloud de AWS que seleccione.
La seguridad y la normativa es una responsabilidad compartida entre AWS y el cliente. Este modelo compartido puede ayudar a aliviar la carga operativa del cliente, ya que AWS opera, gestiona y controla los componentes, desde el sistema operativo del host y la capa de virtualización hasta la seguridad física de las instalaciones en las que opera el servicio.
**Parches y actualizaciones del sistema operativo de una instancia de Amazon Elastic Compute Cloud (Amazon EC2)**
La seguridad "en" el Cloud es responsabilidad del cliente. Un servicio como Amazon Elastic Compute Cloud (Amazon EC2) está categorizado como Infraestructura como Servicio (IaaS) y, como tal, requiere que el cliente realice todas las tareas necesarias de configuración y gestión de la seguridad.
**Habilitación del cifrado de datos almacenados en buckets de Amazon Simple Storage Service (Amazon S3)**
En el modelo de responsabilidad compartida de AWS, los clientes son responsables de administrar sus datos (incluidas las opciones de cifrado), clasificar sus activos y utilizar las herramientas de IAM para aplicar los permisos adecuados.Visión general del modelo de responsabilidad compartida de AWS:  vía - https://aws.amazon.com/compliance/shared-responsibility-model/
Opciones incorrectas:
**Seguridad global de la red de AWS** - La gestión de la infraestructura del Cloud es responsabilidad de AWS.
**Garantizar que los empleados de AWS no puedan acceder a los datos de los clientes** - Garantizar la protección de los datos de los clientes y mantenerlos a salvo de los empleados de AWS es responsabilidad de AWS.
**Validación de normativa de la infraestructura del Cloud** - La seguridad y la normativa en el Cloud son responsabilidad de AWS.
Referencia:
https://aws.amazon.com/compliance/shared-responsibility-model/

Pregunta 4: **OmitidoUna empresa de productos sanitarios busca una forma duradera y rentable de almacenar sus datos históricos. Debido a requisitos de normativa, los datos deben almacenarse durante 10 años. ¿Qué solución de almacenamiento de AWS sugerirías?**
• **Amazon Elastic File System (Amazon EFS)**
• **Amazon S3 Glacier Flexible Retrieval**
• **Amazon S3 Glacier Deep Archive(Correcto)**
• **AWS Storage Gateway
Explicación**
Opción correcta:
**Amazon S3 Glacier Deep Archive**
Amazon S3 Glacier Deep Archive es la clase de almacenamiento de menor coste de Amazon S3 y soporta la retención a largo plazo y la preservación digital de datos a los que se puede acceder una o dos veces al año. Está diseñado para clientes -especialmente los de sectores muy regulados, como los Servicios Financieros, la Sanidad y el Sector Público- que conservan conjuntos de datos durante 7-10 años o más para cumplir los requisitos normativos. Amazon S3 Glacier Deep Archive también puede utilizarse para casos de uso de backup y recuperación de desastres. Tiene un tiempo de recuperación (latencia del primer byte) de 12 a 48 horas.Visión general del Amazon S3 Glacier Deep Archive:  vía - https://aws.amazon.com/s3/storage-classes/
Opciones incorrectas:
**Amazon S3 Glacier Flexible Retrieval** - Amazon S3 Glacier Flexible Retrieval (antes S3 Glacier) es la clase de almacenamiento ideal para archivar datos que no requieren acceso inmediato, pero que necesitan flexibilidad para recuperar grandes conjuntos de datos sin coste alguno, como los casos de uso de backup o recuperación de desastres. Amazon S3 Glacier Deep Archive es más adecuado, ya que es más rentable que la Amazon S3 Glacier Flexible Retrieval para un caso de uso determinado.
**AWS Storage Gateway** - AWS Storage Gateway es un servicio híbrido de almacenamiento en el Cloud que te proporciona acceso local a un almacenamiento prácticamente ilimitado en el Cloud. Todos los datos transferidos entre la puerta de enlace y el almacenamiento de AWS se cifran mediante SSL (para los tres tipos de puertas de enlace: de archivos, de volumen y de cinta). AWS Storage Gateway no puede utilizarse para archivar datos.
**Amazon Elastic File System (Amazon EFS)** - Amazon Elastic File System (Amazon EFS) proporciona un sistema de archivos NFS elástico, sencillo, escalable y totalmente administrado para su uso con los servicios en el Cloud de AWS y los recursos locales. Está diseñado para escalar bajo demanda a petabytes sin interrumpir las aplicaciones, creciendo y reduciéndose automáticamente a medida que añades y eliminas archivos, eliminando la necesidad de aprovisionar y administrar la capacidad para acomodar el crecimiento.
Referencia:
https://aws.amazon.com/s3/storage-classes/

Pregunta 5: **OmitidoUna empresa de TI ha desplegado un sitio web estático en Amazon Simple Storage Service (Amazon S3), pero el sitio web sigue siendo inaccesible. Como Cloud Practioner, ¿cuál de las siguientes soluciones sugerirías para resolver este problema?**
• **Desactivar el cifrado de Amazon S3**
• **Habilitar la replicación de Amazon S3**
• **Corregir la política de bucket de Amazon S3(Correcto)**
• **Habilitar el versionado de Amazon S3
Explicación**
Opción correcta:
**Corregir la política de bucket de Amazon S3**
Para alojar un sitio web estático en Amazon S3, debes configurar un bucket de Amazon S3 para el alojamiento de sitios web y, a continuación, cargar el contenido de tu sitio web en el bucket. Cuando configuras un bucket como sitio web estático, debes habilitar el alojamiento de sitios web, establecer permisos y crear y añadir un documento índice.Alojar un sitio web estático en Amazon S3:  vía - https://docs.aws.amazon.com/AmazonS3/latest/dev/WebsiteHosting.html
Si quieres configurar un bucket existente como sitio web estático de acceso público, debes editar los ajustes de bloqueo de acceso público para ese bucket. Amazon S3 aplica la combinación más restrictiva de los ajustes de acceso público en bloque a nivel de bucket y a nivel de cuenta.A continuación te explicamos cómo puedes editar la configuración de acceso público para los buckets de S3:  vía - https://docs.aws.amazon.com/AmazonS3/latest/dev/WebsiteAccessPermissionsReqd.html

**Desactivar el cifrado de Amazon S3**
**Habilitar el versionado de Amazon S3**
**Habilitar la replicación de Amazon S3**
Desactivar el cifrado de Amazon S3, activar el versionado de Amazon S3 o activar la replicación de Amazon S3 no tienen ninguna relación con el despliegue de un sitio web estático en Amazon S3, por lo que estas opciones no son correctas.

Pregunta 6: **Omitido¿Qué servicio de AWS utilizarás para conectar de forma privada tu nube privada virtual (VPC) a Amazon Simple Storage Service (Amazon S3)?**
• **AWS Direct Connect**
• **Amazon API Gateway**
• **VPC Endpoint Gateway(Correcto)**
• **AWS Transit Gateway
Explicación**
Opción correcta:
**VPC Endpoint Gateway**
Un endpoint de VPC te permite conectar de forma privada tu VPC a los servicios de AWS compatibles y a los servicios de endpoint de VPC impulsados por AWS PrivateLink sin necesidad de una puerta de enlace a Internet, un dispositivo NAT, una conexión VPN o una conexión AWS Direct Connect. Las instancias de tu VPC no necesitan direcciones IP públicas para comunicarse con los recursos del servicio. El tráfico entre tu VPC y el otro servicio no sale de la red de Amazon.
Hay dos tipos de enpoints de la VPC: Interface VPC Endpoints (endpoints de interfaz) y VPC Endpoint Gateway (endpoints de puerta de enlace / Gateway).
Un endpoint de interfaz es una interfaz de red elástica con una dirección IP privada del rango de direcciones IP de tu subred que sirve como punto de entrada para el tráfico destinado a un servicio soportado. Los endpoints de interfaz funcionan con AWS PrivateLink, una tecnología que te permite acceder de forma privada a los servicios utilizando direcciones IP privadas.
Un VPC Endpoint Gateway es una puerta de enlace que especificas como destino de una ruta en tu tabla de enrutamiento (tabla de rutas) para el tráfico destinado a un servicio de AWS soportado. Se admiten los siguientes servicios de AWS:
• Amazon Simple Storage Service (Amazon S3)
• Amazon DynamoDB
Alerta de examen:
Puede que veas una pregunta sobre este concepto en el examen. Recuerda que sólo Amazon S3 y Amazon DynamoDB soportan el endpoint de gateway de VPC. Todos los demás servicios que admiten endpoints de VPC utilizan un endpoint de interfaz de VPC (ten en cuenta que Amazon S3 también admite el endpoint de interfaz de VPC).
Opciones incorrectas:
**AWS Direct Connect** - AWS Direct Connect es una solución de servicio en el Cloud que facilita el establecimiento de una conexión de red dedicada desde tus instalaciones a AWS. Puedes utilizar AWS Direct Connect para establecer una interfaz virtual privada desde tu red local directamente a tu Amazon VPC. Esta conexión privada tarda al menos un mes en completarse.
**AWS Transit Gateway** - AWS Transit Gateway conecta las VPC y las redes locales a través de un hub central. Esto simplifica tu red y pone fin a las complejas relaciones de interconexión. Actúa como un router en el Cloud: cada nueva conexión sólo se realiza una vez. Este servicio es útil para reducir la compleja topología del VPC Peering cuando intervienen muchos sistemas.
**Amazon API Gateway** - Amazon API Gateway es un servicio totalmente administrado que facilita a los desarrolladores la creación, publicación, mantenimiento, monitorización y seguridad de API a cualquier escala. Las API actúan como la "puerta de entrada" para que las aplicaciones accedan a los datos, la lógica empresarial o la funcionalidad de tus servicios backend.
Referencia:
https://docs.aws.amazon.com/vpc/latest/userguide/vpc-endpoints.html

Pregunta 7: **OmitidoUna agencia de ciberseguridad utiliza el Cloud de AWS y quiere llevar a cabo evaluaciones de seguridad en su propia infraestructura de AWS sin ninguna aprobación previa de AWS. ¿Cuál de las siguientes opciones describe/facilita esta práctica?**
• **Pruebas de penetración(Correcto)**
• **Amazon Inspector**
• **AWS Secrets Manager**
• **Prueba de estrés de la red
Explicación**
Opción correcta:
**Pruebas de penetración**
Los clientes de AWS pueden llevar a cabo evaluaciones de seguridad o pruebas de penetración contra su infraestructura de AWS sin aprobación previa para unos pocos servicios comunes de AWS. Los clientes no pueden realizar evaluaciones de seguridad de la infraestructura de AWS, ni de los propios servicios de AWS.
Opciones incorrectas:
**Prueba de estrés de la red** - AWS considera "prueba de estrés de red" cuando una prueba envía un gran volumen de tráfico legítimo o de prueba a una aplicación de destino específica. Se espera que el endpoint y la infraestructura sean capaces de manejar este tráfico.
**Amazon Inspector** - Amazon Inspector es un servicio automatizado de evaluación de la seguridad que te ayuda a comprobar la accesibilidad no intencionada a la red de tus instancias de Amazon EC2 y las vulnerabilidades de esas instancias de Amazon EC2. Las evaluaciones de Amazon Inspector se te ofrecen como paquetes de reglas predefinidas asignadas a las mejores prácticas comunes de seguridad y a las definiciones de vulnerabilidades.
**AWS Secrets Manager** - AWS Secrets Manager te ayuda a proteger los secretos necesarios para acceder a tus aplicaciones, servicios y recursos informáticos. El servicio te permite rotar, administrar y recuperar fácilmente credenciales de bases de datos, claves API y otros secretos a lo largo de su ciclo de vida. Los usuarios y las aplicaciones recuperan los secretos con una llamada a las API del AWS Secrets Manager, eliminando la necesidad de codificar información confidencial en texto plano.
Referencia:
https://aws.amazon.com/security/penetration-testing/

Pregunta 8: **OmitidoUna empresa de TI tiene una arquitectura de nube híbrida y quiere centralizar los registros de servidor de sus instancias de Amazon Elastic Compute Cloud (Amazon EC2) y de sus servidores locales. ¿Cuál de las siguientes opciones es la MÁS eficaz para este caso de uso?**
• **Utiliza Amazon CloudWatch Logs tanto para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) como para los servidores locales(Correcto)**
• **Utiliza Amazon CloudWatch Logs para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y AWS CloudTrail para los servidores locales**
• **Utiliza AWS Lambda para enviar los datos de registro de la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y de los servidores locales a Amazon CloudWatch Logs**
• **Utiliza AWS CloudTrail para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y Amazon CloudWatch Logs para los servidores locales
Explicación**
Opción correcta:
**Utiliza Amazon CloudWatch Logs tanto para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) como para los servidores locales**
Puedes utilizar Amazon CloudWatch Logs para monitorizar, almacenar y acceder a tus archivos de registro desde instancias de Amazon Elastic Compute Cloud (Amazon EC2), AWS CloudTrail, Route 53 y otras fuentes como servidores locales.
Amazon CloudWatch Logs te permite centralizar los logs de todos tus sistemas, aplicaciones y servicios de AWS que utilices, en un único servicio altamente escalable. Luego puedes verlos fácilmente, buscar en ellos códigos de error o patrones específicos, filtrarlos en función de campos concretos o archivarlos de forma segura para futuros análisis.
Opciones incorrectas:
**Utiliza AWS Lambda para enviar los datos de registro de la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y de los servidores locales a Amazon CloudWatch Logs**
AWS Lambda te permite ejecutar código sin aprovisionar ni administrar servidores. Sólo pagas por el tiempo de computación que consumes. Lambda no puede utilizarse para centralizar los registros de las instancias de Amazon Elastic Compute Cloud (Amazon EC2) y los servidores locales.
**Utiliza Amazon CloudWatch Logs para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y AWS CloudTrail para los servidores locales**
**Utiliza AWS CloudTrail para la instancia de Amazon Elastic Compute Cloud (Amazon EC2) y Amazon CloudWatch Logs para los servidores locales**
AWS CloudTrail es un servicio que permite la gobernanza, la normativa, la auditoría operativa y la auditoría de riesgos de tu cuenta de AWS. Con AWS CloudTrail, puedes registrar, monitorizar continuamente y conservar la actividad de la cuenta relacionada con acciones en toda tu infraestructura de AWS. AWS CloudTrail proporciona un historial de eventos de la actividad de tu cuenta de AWS, incluidas las acciones realizadas a través de la consola de administración de AWS, los SDK de AWS, las herramientas de línea de comandos y otros servicios de AWS. AWS CloudTrail no puede utilizarse para centralizar los registros de servidor de las instancias de Amazon Elastic Compute Cloud (Amazon EC2) ni de los servidores locales, por lo que ambas opciones son incorrectas.
Referencias:
https://docs.aws.amazon.com/AmazonCloudWatch/latest/logs/WhatIsCloudWatchLogs.html
https://docs.aws.amazon.com/AmazonCloudWatch/latest/logs/AgentReference.html

Pregunta 9: **Omitido¿Cuál de las siguientes es una parte de la Infraestructura Global de AWS?**
• **Virtual Private Network (VPN)**
• **Virtual Private Cloud (VPC)**
• **Región de AWS(Correcto)**
• **Subred
Explicación**
Opción correcta:
**Región de AWS**
Una región de AWS es una ubicación física en todo el mundo donde AWS construye sus centros de datos. Cada grupo de centros de datos lógicos se denomina zona de disponibilidad (AZ). Cada región de AWS consta de múltiples AZ aisladas y físicamente separadas dentro de un área geográfica.Consulta esta ilustración para conocer las regiones de AWS en EE.UU:  vía - https://aws.amazon.com/about-aws/global-infrastructure/regions_az/
Opciones incorrectas:
**Virtual Private Cloud (VPC)** - Amazon Virtual Private Cloud (Amazon VPC) es una sección lógicamente aislada del Cloud de AWS donde puedes lanzar recursos de AWS en una red virtual que tú defines. Tienes un control total sobre tu entorno de red virtual, incluida la selección de tu rango de direcciones IP, la creación de subredes y la configuración de tablas de rutas y puertas de enlace de red. Una VPC abarca todas las zonas de disponibilidad de la región.
**Virtual Private Network (VPN)** - AWS Virtual Private Network (AWS VPN) te permite establecer un túnel cifrado seguro y privado desde tu red local a la red global de AWS. AWS VPN se compone de dos servicios: VPN Site-to-Site de AWS y VPN de cliente de AWS.
**Subred** - Una subred es un rango de direcciones IP dentro de tu VPC. Una subred abarca sólo una zona de disponibilidad de la región.
Estas tres opciones no forman parte de la Infraestructura Global de AWS.
Referencia:
https://aws.amazon.com/about-aws/global-infrastructure/regions_az/

Pregunta 10: **OmitidoUna empresa busca una ruta guiada que le ayude a implementar, configurar y proteger sus nuevas cargas de trabajo, al tiempo que se asegura de que está preparada para operaciones continuas en el Cloud. ¿Cuál de los siguientes servicios/herramientas de AWS puede aprovecharse para este caso de uso?**
• **Modelo de responsabilidad compartida de AWS**
• **Fundamentos del Cloud(Correcto)**
• **AWS Config**
• **AWS Trusted Advisor
Explicación**
Opción correcta:
**Fundamentos del Cloud**
Cloud Foundations proporciona una ruta guiada para ayudar a los clientes a implementar, configurar y proteger sus nuevas cargas de trabajo, garantizando al mismo tiempo que están preparados para las operaciones continuas en el Cloud. Cloud Foundations ayuda a los clientes a navegar a través de las decisiones que necesitan tomar mediante Servicios de AWS, Soluciones de AWS, Soluciones de socios y Orientación.Fundamentos del Cloud:  vía - https://aws.amazon.com/architecture/cloud-foundations/
Opciones incorrectas:
**AWS Trusted Advisor** - AWS Trusted Advisor es una herramienta online que proporciona orientación en tiempo real para ayudar a aprovisionar tus recursos siguiendo las mejores prácticas de AWS. Ya sea para establecer nuevos flujos de trabajo, desarrollar aplicaciones o como parte de una mejora continua, las recomendaciones proporcionadas por Trusted Advisor ayudan regularmente a mantener tus soluciones aprovisionadas de forma óptima. AWS Trusted Advisor analiza tu entorno de AWS y proporciona recomendaciones de mejores prácticas en cinco categorías: Optimización de costes, Rendimiento, Seguridad, Tolerancia a fallos, Límites de servicio.
**AWS Config** - AWS Config es un servicio que te permite evaluar, auditar y valorar las configuraciones de tus recursos de AWS. Con AWS Backup, sólo pagas por la cantidad de almacenamiento de copia de seguridad que utilizas y por la cantidad de datos de copia de seguridad que restauras en el mes. Pregunta 28: **OmitidoLas políticas de AWS Identity and Access Management (AWS IAM) se escriben como documentos JSON. ¿Cuáles de los siguientes son elementos obligatorios de una política IAM?**

• **Effect, Action(Correcto)**
• **Effect, Sid**
• **Sid, Principal
Explicación**
Opción correcta:
**Effect, Action**
La mayoría de las políticas se almacenan en AWS como documentos JSON. Las políticas basadas en la identidad y las políticas utilizadas para establecer límites de permisos son documentos de política JSON que adjuntas a un usuario o rol. Las políticas basadas en recursos son documentos de política JSON que adjuntas a un recurso.
Un documento de política JSON incluye estos elementos:
1. Información opcional sobre toda la política en la parte superior del documento
2. Una o más declaraciones individuales
Cada declaración incluye información sobre un único permiso. La información de una declaración está contenida en una serie de elementos.
1. Versión - Especifica la versión del lenguaje de la política que quieres utilizar. Como mejor práctica, utiliza la última versión de 2012-10-17.
2. Declaración - Utiliza este elemento principal de la política como contenedor de los siguientes elementos. Puedes incluir más de una declaración en una política.
    1. Sid (Opcional) - Incluye un ID de declaración opcional para diferenciar tus declaraciones.
    2. Effect - Utiliza ALLOW o DENY para indicar si la política permite o deniega el acceso.
    3. Principal (Obligatorio sólo en algunas circunstancias) - Si creas una política basada en recursos, debes indicar la cuenta, usuario, rol o usuario federado al que quieres permitir o denegar el acceso. Si estás creando una política de permisos IAM para adjuntarla a un usuario o rol, no puedes incluir este elemento. El principal está implícito como ese usuario o rol.
    4. Action - Incluye una lista de acciones que la política permite o deniega.
    5. Resource (Obligatorio sólo en algunas circunstancias) - Si creas una política de permisos IAM, debes especificar una lista de recursos a los que se aplican las acciones. Si creas una política basada en recursos, este elemento es opcional. Si no incluyes este elemento, el recurso al que se aplica la acción es el recurso al que se adjunta la política.
    6. Condition (Opcional) - Especifica las circunstancias en las que la política concede el permiso.
Opciones incorrectas:
**Sid, Principal**
**Action, Condition**
**Effect, Sid**
Estas tres opciones contradicen la explicación anterior, por lo que son incorrectas.

Opción correcta:
**Almacenamiento de archivos en una ubicación accesible para satisfacer los requisitos de auditoría**
La clase de almacenamiento Amazon EFS Standard-Infrequent Access (EFS Standard-IA) reduce los costes de almacenamiento de los archivos a los que no se accede todos los días. AWS recomienda la clase de almacenamiento Amazon EFS Standard-Infrequent Access (EFS Standard-IA) si necesitas que todo tu conjunto de datos sea fácilmente accesible y quieres ahorrar automáticamente en costes de almacenamiento para los archivos a los que se accede con menos frecuencia. Los ejemplos incluyen mantener los archivos accesibles para satisfacer requisitos de auditoría, realizar análisis históricos o realizar copias de seguridad y recuperación. El almacenamiento de Amazon EFS Standard-Infrequent Access (EFS Standard-IA) es compatible con todas las características de Amazon EFS, y está disponible en todas las regiones de AWS en las que está disponible Amazon EFS.

**Almacenamiento de objetos para cargas de trabajo que necesitan velocidades de latencia inferiores al segundo para acceder a los datos** - Amazon EFS es un servicio de sistema de archivos y no un servicio de almacenamiento de objetos. Deberías utilizar Amazon S3 para el almacenamiento de objetos. Por tanto, esta opción es incorrecta.
**Uso como volumen de arranque para instancias de alta disponibilidad de Amazon Elastic Compute Cloud (Amazon EC2)** - Amazon EFS no puede utilizarse como volumen de arranque para instancias de Amazon Elastic Compute Cloud (Amazon EC2). Para los volúmenes de arranque se utilizan los volúmenes de Amazon Elastic Block Storage (Amazon EBS).

Pregunta 64: **OmitidoUn grupo de investigación quiere aprovisionar una instancia de Amazon Elastic Compute Cloud (Amazon EC2) para una aplicación flexible que puede interrumpirse. Como Cloud Practitioner, ¿cuál de las siguientes recomendarías como la opción MÁS óptima en cuanto a costes?**
• **Instancia bajo demanda**
• **Instancia spot(Correcto)**
• **Instancia reservada (RI)**
• **Host dedicado
Explicación**
Opción correcta:
**Instancia spot**
Una instancia spot es una instancia EC2 no utilizada que está disponible por menos del precio bajo demanda. Como las instancias de subasta te permiten solicitar instancias EC2 no utilizadas con grandes descuentos (hasta el 90%), puedes reducir significativamente tus costes de Amazon EC2. Las Instancias de subasta son muy adecuadas para análisis de datos, trabajos por lotes, procesamiento en segundo plano y otras tareas flexibles que pueden interrumpirse. Pueden interrumpirse con poca antelación, por lo que no son adecuadas para cargas de trabajo críticas que deban ejecutarse en un momento determinado.Visión general de las opciones de precios de Amazon EC2:  vía - https://aws.amazon.com/ec2/pricing/
Opciones incorrectas:
**Instancia bajo demanda** - Una instancia bajo demanda es una instancia que utilizas bajo demanda. Tienes pleno control sobre su ciclo de vida: tú decides cuándo lanzarla, detenerla, hibernarla, iniciarla, reiniciarla o terminarla. No es necesario un compromiso a largo plazo cuando adquieres Instancias Bajo Demanda. No hay que pagar nada por adelantado y sólo pagas por los segundos que se ejecutan tus Instancias Bajo Demanda. El precio por segundo de ejecución de una Instancia bajo demanda es fijo. Las instancias bajo demanda no se pueden interrumpir. Sin embargo, las instancias bajo demanda no son tan rentables como las instancias spot, por lo que esta opción no es correcta.
**Instancia reservada (RI)** - Las instancias reservadas (RI) te proporcionan un ahorro significativo (hasta el 75%) en tus costes de Amazon EC2 en comparación con los precios de las Instancias Bajo Demanda. Las Instancias reservadas (RI) no son instancias físicas, sino un descuento de facturación aplicado al uso de Instancias bajo demanda en tu cuenta. Puedes adquirir una instancia reservada (RI) para un compromiso de un año o de tres años, siendo el compromiso de tres años el que ofrece un mayor descuento. Las instancias reservadas (RI) no pueden interrumpirse. Las instancias reservadas (RI) no son tan rentables como las instancias bajo demanda, por lo que esta opción no es correcta.
**Host dedicado** - Los hosts dedicados de Amazon EC2 te permiten utilizar tus licencias de software elegibles de proveedores como Microsoft y Oracle en Amazon EC2 para que obtengas la flexibilidad y rentabilidad de utilizar tus licencias, pero con la resistencia, sencillez y elasticidad de AWS. Un host dedicado de Amazon EC2 es un servidor físico totalmente dedicado para tu uso, de modo que puedes ayudar a cumplir los requisitos corporativos. No son rentables en comparación con las instancias spot. Así que esta opción no es correcta.

https://aws.amazon.com/ec2/pricing/
